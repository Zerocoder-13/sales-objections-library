# Инструкция пользователя — Библиотека возражений и кейсов для B2B-продаж

## 1. Структура таблицы

Основной файл — `sales-objections-library.md` (открывается в Obsidian). Колонки:

| Колонка | Описание |
|---|---|
| № | номер строки |
| Возражение | формулировка близко к тому, как говорит клиент |
| Контекст / когда возникает | на каком этапе переговоров встречается |
| Ответ / аргумент | что отвечать клиенту |
| Кейс / цифра | реальный пример с результатом |
| Источник | ссылка на кейс, файл в `cases/` или «личный опыт менеджера» |

**Поиск возражения:** открыть файл в Obsidian, `Ctrl+F`, ввести ключевые слова из возражения клиента (например «бюджет», «время», «руководство»).

**Использование на звонке/в КП:** найти подходящую строку, использовать готовый «Ответ/аргумент» и подкрепить его «Кейс/цифра» со ссылкой на «Источник» (можно открыть файл в `cases/`, если он есть).

## 2. Веб-интерфейс

Файл `2026-06-15-objections-library-viewer.html` — наглядная альтернатива таблице: открывается в любом браузере двойным кликом, не требует установки и интернета (все данные встроены в страницу).

### Общий вид

![Общий вид веб-интерфейса](screenshots/viewer-overview.png)

Вверху — заголовок, строка поиска, кнопки фильтров по компаниям и кнопка «+ Добавить возражение». Ниже — список всех возражений в виде разворачиваемых карточек.

### Поиск

Введите слово или фразу в строку поиска — подходящие строки автоматически раскрываются, а совпадения подсвечиваются жёлтым.

![Поиск по слову «бюджет»](screenshots/viewer-search-budget.png)

### Фильтр по компании

Нажмите на кнопку с названием компании (например «Инвитро») — останутся только возражения, у которых есть кейс этой компании. Кнопка «Все» сбрасывает фильтр.

![Фильтр по компании «Инвитро»](screenshots/viewer-filter-invitro.png)

### Разворачивание карточки

Клик по заголовку возражения раскрывает карточку с полями «Контекст», «Ответ/аргумент», «Кейс/цифра» и «Источник». Если для строки есть развёрнутый файл кейса — появляется кнопка «Открыть файл кейса →».

![Развёрнутая карточка возражения №2](screenshots/viewer-item-expanded.png)

### Добавление нового возражения через форму

Кнопка «+ Добавить возражение» открывает форму. Заполните поля (обязательное — только «Возражение») и нажмите «Добавить».

![Форма добавления нового возражения](screenshots/viewer-add-modal.png)

Новая запись сохраняется **только в браузере** (localStorage) с пометкой «Добавлено вами» и появляется в списке с дополнительными кнопками:

![Добавленная запись с пометкой «Добавлено вами»](screenshots/viewer-custom-item-expanded.png)

- **«Скопировать как строку таблицы»** — копирует запись в формате строки markdown-таблицы. Передайте этот текст агенту в Claude Code для проверки и подтверждения перед внесением в `sales-objections-library.md`.
- **«Удалить»** — убирает черновик из браузера.

> Запись из веб-интерфейса не попадает в общую библиотеку автоматически — это черновик для согласования с агентом, как и любое другое изменение.

## 3. Как пополнять библиотеку (через Claude Code)

1. **Новое возражение.** Когда на звонке встречается формулировка, которой нет в таблице — описать её агенту своими словами (или скопировать черновик из веб-интерфейса).
2. **Поиск похожего.** Агент ищет похожую строку в таблице. Если похожая есть — обсуждается, дополнить существующую строку или завести новую.
3. **Кейс и источник.** Если подходящий кейс уже есть в библиотеке (личный опыт, ранее найденный материал) — агент предлагает его. Если нет — менеджер явно говорит «запусти ресёрч», и агент ищет подтверждающий кейс.
4. **Формулировка.** Агент предлагает текст строки: возражение, контекст, ответ/аргумент, кейс/цифра, источник.
5. **Подтверждение.** Менеджер подтверждает формулировку или просит скорректировать. Без подтверждения в таблицу ничего не вносится.
6. **Запись.** Агент добавляет/обновляет строку в `sales-objections-library.md`. При необходимости — создаёт отдельный файл кейса в `cases/`.

**Шаблон новой записи:** `templates/new-objection-row-template.md` — формат строки + чек-лист перед добавлением. В Obsidian: `Settings → Templates` → указать папку `templates`, затем `Ctrl+P → Insert template`.

## 4. Очередь тем для ресёрча — `research-queue.md`

Список открытых тем: что ещё нужно найти, чтобы усилить или дополнить существующие строки таблицы.

- **«открыто»** — тема ждёт ресёрча
- **«частично закрыто»** — часть данных уже есть, нужны дополнительные
- После внесения результата задача переносится в «Архив выполненных» с датой и ссылкой на источник

### ⚠️ Главное правило — ресёрч только по запросу

Агент **никогда** не запускает ресёрч автоматически, по расписанию или в фоне. Любой запуск исследования — только после явной команды менеджера, например «запусти ресёрч по теме 2 из очереди» или «найди кейс про X».

### Рекомендуемая периодичность

| Когда | Что делать |
|---|---|
| После каждой сделки | Проверить, не возникло ли новое возражение, которого нет в таблице. Если да — пройти шаги 1-6 (обычно ~1 минута работы агента) |
| Раз в 1-2 недели (по желанию) | Запустить ресёрч по пункту 2 очереди («Мониторинг новых кейсов Zerocoder») |
| Раз в месяц (по желанию) | Запустить ресёрч по пунктам 3-4 очереди (отраслевая статистика ROI, сравнение с бесплатными материалами) — усиление аргументов №1, №4, №8, №10 |
| По необходимости | Если клиент привёл новый аргумент или появился новый продукт/кейс Zerocoder — добавить тему в `research-queue.md` или сразу попросить агента найти материал |

## 5. CSV-экспорты — `exports/`

Снимки данных для импорта в Google Таблицы: база возражений и кейсов, полное описание проекта, эта инструкция.

- **Когда обновлять:** после значимых изменений в таблице (новые строки, изменения статусов) — попросить агента обновить `exports/2026-06-15-objections-cases-database.csv`
- **Как импортировать:** в Google Таблицах — `Файл → Импорт → Загрузка` → выбрать CSV → «Заменить лист» или «Вставить новый лист»
