# Как менеджер по продажам автоматизировал базу возражений с Claude Code за 30 дней

## Кто клиент

Менеджер отдела B2B-продаж университета Zerocoder — много лет в продажах, но привык постоянно осваивать новые инструменты для работы с клиентами.

## Проблема

В Obsidian была заготовка библиотеки возражений — таблица из 12 формулировок («Дорого», «Нет времени», «Уже есть свой отдел обучения» и т.д.) с готовыми ответами, но колонка «Кейс/цифра» во всех строках стояла как «заполнить». На звонках аргументы были, а подтверждающих кейсов и цифр под рукой не было.

## Решение

Без платных подписок — только Claude Code Pro и бесплатные инструменты. Агент:
- провёл ресёрч кейсов обучения Zerocoder в открытых источниках и сопоставил их с нужными строками таблицы;
- помог оформить реальные истории из личной практики менеджера в формате готовых кейсов;
- собрал отдельные файлы кейсов (Инвитро, ИИ-разработчики и др.) в папке `cases/`;
- настроил очередь тем для дальнейшего ресёрча (`research-queue.md`) и шаблон для новых строк — оба запускаются только по запросу менеджера, без автоматики.

## Результат

- За 30 дней колонки «Кейс/цифра» и «Источник» заполнены в 12 из 13 строк — старт был 0 из 12
- Новое возражение с реального звонка обработано целиком (поиск похожего, формулировка ответа, кейс, источник, обновление таблицы) за ~1 минуту работы агента — против 10-15 минут вручную
- Создано 4 файла кейсов в `cases/` для использования в КП и презентациях
- Библиотека стала самообновляемой: новое возражение → строка с готовым аргументом и подтверждённым источником в течение минуты

> Цитата менеджера: *заполнить*

## Итог

Шаблон из 12 пустых строк превратился в рабочий инструмент с реальными кейсами и проверяемыми источниками — без сторонних сервисов, на одном Claude Code Pro. Если у вас есть таблица возражений «на бумаге», но без подтверждённых аргументов — такой же подход поможет довести её до рабочего состояния за месяц.
